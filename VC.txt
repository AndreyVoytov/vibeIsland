Мы работаем над HTML+js игрой в портретной ориентации (в ланшафтной ориентации будут отображаться черные полосы по бокам).
Проект состоит из независимых html файлов управляемых преимущественно js (но возможно с использованием html), которые подключаются в index.html чтобы объедениться в игру. Пиши код всех файлов строго в чате.

Если в каком-то из файлов в игру добавляется графический объект, то он должен быть 1) сэмулирован с помощью графических примитивов 2) добавлен с помощью картинки, с указанием габаритов в пикселях. Выбор 1 или 2 происходит с помощью проверки условия о том, что строка URL содержит подстроку 'images' (если содержит - 2, иначе 1)

Если в каком-то из файлов в игру добавляется htlm-элементы интерфейса, панели, кнопки - они должны отрисовываться каждая в отдельном методе 

Всё позиционирование графических объектов должно быть в % от ширины канваса


Сгенерируй новую механику по инструкции на картинке, по возможности не внося измененений в уже существующие файлы
---------------------------------------


Сгенерируй следующие файлы: 

layout.html - синее поле и посередине зеленый квадратный остров со скругленными краями. Добавь чуть видный обрыв на торцевой части, для псевдо-изометрического эффекта.
Также файл сетает в localStorage.map карту проходимости (0 - море, непроходимо, 1 - остров, проходимо) в расчете что экран игры состоит из 192х108  клеток

character.html => добавляет на страницу персожа и классический мобильный джойстик из двух кружков для управления этим персонажем. Персонаж только плавно бегает по клеткам, которым в localStorage.map соответствует цифра 1, а в остальные - упирается и отбрасывается назад (надо оптимизировать проверку, что пресонаж уперся, чтобы не проверять все 192х108 клеток )

index.html, который простейшим образом подкючает эти файлы для объединения их логики.
